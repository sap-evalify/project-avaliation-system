### Sistema de Avaliação de Projetos da Fabrica de Software

## Objetivo
Os problemas atuais na gestão de projetos acadêmicos

Dificuldade no Acompanhamento - Professores gerenciam múltiplas equipes simultaneamente, sem uma visão exata de progresso.

Falta de Centralização - Informações de projetos, artefatos, equipes e notas ficam espalhadas em planilhas, e-mails e pastas.

Ausência de Feedback 360° - Alunos não têm um canal estruturado para avaliar seus próprios colegas (peer-to-peer), uma etapa crucial para o desenvolvimento de soft skills.

Nosso sistema foi projetado para resolver exatamente esses e mais pontos.

## Artefatos

Ao final, o repositório deverá possuir o seguinte conteúdo:
 
- Diagrama de Casos de Uso

- Diagrama de Classes 

- Diagrama de Atividades

- Diagrama de Sequências
